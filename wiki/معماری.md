# معماری

کسب‌یار یک monorepo سه‌لایه است.

## ساختار مخزن

```
apps/web          اپ اصلی — Next.js 15 + لندینگ
apps/ai-service   سرویس دستیار — FastAPI
packages/shared   جلالی، فرمت، billing، observability
packages/ui       اجزای UI مشترک
prisma/           schema، migration، seed
docs/             مستندات عمیق مهندسی
```

## لایه‌ها

1. **Presentation** — App Router، RTL فارسی  
2. **Application** — سرویس‌های دامنه در `apps/web/src/server`  
3. **Infrastructure** — Prisma/PostgreSQL، FastAPI، Docker  

## جریان داده

```
UI (Server/Client Components)
  → API Routes / Server Services
    → Prisma → PostgreSQL
    → AI Client → FastAPI
```

## هسته در برابر بسته عمودی

- هسته: احراز هویت، سازمان، CRM، فاکتور، پرداخت، وظیفه، گزارش، فایل، audit  
- عمودی: کلینیک / خرده‌فروشی / سفر — [بسته‌های صنفی](بسته‌های-صنفی)
