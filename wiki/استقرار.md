# استقرار

## Vercel (وب)

نسخه فعلی: https://kasbyar.vercel.app

### تنظیمات پروژه

| فیلد | مقدار |
|------|--------|
| Root Directory | `apps/web` |
| Include files outside root | ✅ فعال |
| Framework | Next.js |

`apps/web/vercel.json` دستور install/build monorepo را تنظیم می‌کند.

### متغیرهای الزامی

| متغیر | نمونه |
|--------|--------|
| `DATABASE_URL` | Neon / Postgres |
| `SESSION_SECRET` | رشته تصادفی ≥ ۳۲ کاراکتر |
| `NEXT_PUBLIC_APP_URL` | `https://kasbyar.vercel.app` |
| `APP_ENV` | `production` |
| `ALLOW_SEED` | `false` |

فهرست کامل: [متغیرهای محیط](./متغیرهای-محیط.md)

### دیتابیس اولیه

پس از داشتن `DATABASE_URL`:

```bash
DATABASE_URL="postgresql://..." npm run db:push
# seed فقط برای staging/demo
```

### محدودیت‌های Vercel

| موضوع | توضیح |
|--------|--------|
| آپلود فایل | filesystem موقت — برای production جدی از Blob/S3 |
| AI service | روی Vercel اجرا نمی‌شود؛ جدا deploy کنید |
| backup | خارج از Vercel (اسکریپت/CI) |

Push به `main` معمولاً deploy خودکار را تریگر می‌کند.

## Docker / VPS

برای استقرار کامل‌تر (وب + Postgres + AI) از `docs/DEPLOYMENT.md` و `docker/docker-compose.yml` استفاده کنید.

## CI

Workflow GitHub Actions روی `main`: lint/test/build وب، validate سرویس AI، و build ایمیج‌های Docker.
