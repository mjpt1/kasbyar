# بسته‌های صنفی (Vertical Packs)

هر سازمان یک `industryPack` و اختیاری یک `industrySpecialty` دارد. **هسته مشترک** همیشه فعال است؛ ماژول عمودی فقط وقتی pack مربوطه روشن باشد در منو و API دیده می‌شود. تخصص‌ها پیشخوان `/v/{id}` و برچسب منو می‌سازند.

```
┌─────────────────────────────────────────┐
│           هسته مشترک                     │
│  مشتری · لید · فاکتور · پرداخت · وظیفه  │
└──────────────────┬──────────────────────┘
                   │
     ┌─────────────┼─────────────┐
     ▼             ▼             ▼
  GENERAL      CLINIC      TRAVEL_AGENCY …
  (تخصص‌ها)   (بیمار)         (رزرو)
```

## عمومی (`GENERAL`) + تخصص‌ها

بستهٔ عمومی ماژول عمودی جدا ندارد، ولی تخصص‌های زیر را پشتیبانی می‌کند:

| تخصص | مسیر | تمرکز |
|------|------|--------|
| general-business | `/v/general-business` | پیشخوان عمومی |
| freelancer | `/v/freelancer` | فریلنسر: کارفرما، فاکتور، فرصت، کارها |
| software-house | `/v/software-house` | شرکت برنامه‌نویسی: مشتری، لید، تحویل |
| consulting-firm | `/v/consulting-firm` | مشاوره |
| online-services | `/v/online-services` | خدمات آنلاین |
| content-creator | `/v/content-creator` | تولید محتوا |

جزئیات جریان انتخاب: [آنبوردینگ و تخصص‌ها](آنبوردینگ-و-تخصص‌ها)

## کلینیک (`CLINIC`)

| قابلیت | توضیح |
|--------|--------|
| بیماران | پروفایل بیمار متصل به مشتری |
| نوبت‌ها | زمان‌بندی و وضعیت |
| داشبورد | نوبت امروز، از دست‌رفته، پیگیری |

مسیرهای UI: `/clinic`, `/clinic/patients`, `/clinic/appointments`

## خرده‌فروشی (`RETAIL`)

| قابلیت | توضیح |
|--------|--------|
| محصولات | کاتالوگ |
| موجودی | ورود/خروج و کنترل کم‌موجودی |
| داشبورد | فروش و وضعیت انبار |

مسیرهای UI: `/retail`, `/retail/products`, `/retail/inventory`

## آژانس مسافرتی (`TRAVEL_AGENCY`)

| قابلیت | توضیح |
|--------|--------|
| مسافران | پروفایل متصل به مشتری |
| رزروها | وضعیت و تاریخ حرکت |
| داشبورد | درخواست باز، اعزام نزدیک، مانده حساب |

مسیرهای UI: `/travel`, `/travel/bookings`

## فعال‌سازی فنی

| لایه | نقش |
|------|-----|
| `Organization.industryPack` | انتخاب pack سازمان |
| Session | انتقال pack به زمینه درخواست |
| `requirePackPage` / `requireApiPack` | قفل صفحه و API |
| منو | آیتم‌های وابسته به pack |

دسترسی به بسته عمودی همچنین تحت تأثیر **طرح اشتراک** است — [اشتراک و صورتحساب](اشتراک-و-صورتحساب).

## تخصص‌های صنفی (100+ داشبورد)

علاوه بر ۱۷ بسته عمودی، هر سازمان می‌تواند یک `industrySpecialty` داشته باشد — مثلاً «داروخانه»، «آرایشگاه مردانه» یا «تعمیرگاه خودرو». هر تخصص:

- پیشخوان اختصاصی در `/v/{specialty-id}` با ویجت‌های CRM + شاخص‌های بسته پایه
- برچسب مشتری متناسب با صنف (بیمار، مهمان، موکل، …)
- سازمان دمو با slug `demo-spec-{id}` برای نمایش فروش

ثبت تخصص از پنل سوپرادمین → سازمان‌ها انجام می‌شود. مجموع **۱۷ بسته + ۸۸+ تخصص = بیش از ۱۰۰ داشبورد** متمایز.
