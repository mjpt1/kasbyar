# متغیرهای محیط

مرجع خلاصه. جزئیات کامل در مخزن: `docs/ENVIRONMENT.md` و `.env.example`.

## وب (`apps/web`)

| متغیر | الزامی | توضیح |
|--------|--------|--------|
| `DATABASE_URL` | بله | اتصال PostgreSQL |
| `SESSION_SECRET` | بله در production | حداقل ۱۶ کاراکتر (توصیه ≥ ۳۲) |
| `NEXT_PUBLIC_APP_URL` | بله در production | URL عمومی اپ |
| `APP_ENV` | توصیه‌شده | `development` / `staging` / `production` / `test` |
| `ALLOW_SEED` | production: `false` | اجازه عملیات seed مخرب |
| `DEMO_MODE` | خیر | حالت نمایش |
| `NEXT_PUBLIC_DEMO_MODE` | خیر | هم‌راستا با DEMO_MODE |
| `AI_SERVICE_URL` | خیر | آدرس FastAPI |
| `AI_SERVICE_TOKEN` | اگر AI دارید | توکن داخلی |
| `UPLOAD_DIR` | خیر | مسیر آپلود (در Vercel موقت) |
| `SENTRY_DSN` | خیر | مانیتورینگ خطا |
| `VAPID_PUBLIC_KEY` | برای Web Push | کلید عمومی VAPID |
| `VAPID_PRIVATE_KEY` | برای Web Push | کلید خصوصی VAPID |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | اختیاری | آینهٔ کلید عمومی برای کلاینت |
| `VAPID_SUBJECT` | توصیه‌شده | مثلاً `mailto:ops@kesbyar.ir` |

تولید کلید: `npx web-push generate-vapid-keys` — جزئیات: [اعلان‌ها](اعلان‌ها)

## سرویس AI (`apps/ai-service`)

| متغیر | توضیح |
|--------|--------|
| `AI_SERVICE_TOKEN` | توکن مشترک با وب |
| `APP_ENV` | محیط اجرا |
| `CORS_ORIGINS` | مبدأهای مجاز |
| `DATABASE_URL` | اختیاری بسته به قابلیت‌ها |

## نکات

- هرگز `.env` را commit نکنید  
- در Vercel متغیرها را برای Production / Preview جدا تنظیم کنید  
- پس از تغییر env حیاتی، redeploy لازم است
