# محدودیت‌های شناخته‌شده — V1

مواردی که عمداً در V1 کامل نشده‌اند یا نیاز به بهبود دارند.

## محصول

| محدودیت | توضیح | اولویت پس از لانچ |
|---------|--------|-------------------|
| فرم‌های یکسان‌سازی‌نشده | همه فرم‌ها هنوز react-hook-form+zod نیستند | متوسط |
| جداول دستی | برخی لیست‌ها DataTable یکپارچه ندارند | پایین |
| export داده | خروجی Excel/CSV self-service نیست | متوسط |
| اعلان real-time | بدون WebSocket/push | پایین |
| چند workspace فعال | فقط یک org در session | طراحی آینده |

## دستیار هوشمند

| محدودیت | توضیح |
|---------|--------|
| بدون تاریخچه پایدار | پیام‌ها با refresh از بین می‌روند |
| بدون markdown غنی | پاسخ plain text |
| وابستگی AI service | حالت degraded در قطعی سرویس |

## فنی

| محدودیت | توضیح |
|---------|--------|
| migration history | ممکن است baseline در حال تکمیل باشد |
| Prisma tenant middleware | انضباط در سرویس‌ها، نه middleware سراسری |
| backup خودکار cloud | اسکریپت pg_dump — زمان‌بندی خارج از repo |

## دسترسی‌پذیری

- بهبود focus و screen reader انجام شده؛ ممیزی رسمی WCAG انجام نشده
- برخی کنترل‌ها هنوز `confirm()` مرورگر برای حذف استفاده می‌کنند

## بازار ایران

- درگاه پرداخت آنلاین متصل نیست (ثبت دستی پرداخت)
- پیامک/واتساپ خودکار در اتوماسیون محدود است

## امنیت

- audit کامل SIEM نیست — `AuditEvent` سطح محصول
- rate limiting سطح edge ممکن است نیاز به تنظیم production داشته باشد

## ثبت مورد جدید

هنگام کشف محدودیت در پایلوت، این فایل را با تاریخ و شناسه مشتری به‌روز کنید.
